## CI/CD (Phase I)

This repo includes `cloudbuild.yaml` for GitHub-triggered deploys to Cloud Run.

Pipeline behavior:

1. Build Docker image tagged with `$COMMIT_SHA`
2. Push image to Artifact Registry (`cloud-run-source-deploy/jobtracker`)
3. Deploy Cloud Run service (`jobtracker`)
4. Update migration job image (`jobtracker-migrate`) to the same image
5. Execute migration job and wait for completion

Recommended trigger:

- Source: GitHub repo
- Branch: `main` (or `dev` while testing)
- Config: `cloudbuild.yaml`

Required IAM for the Cloud Build service account:

- Cloud Run Admin
- Service Account User (runtime service account)
- Artifact Registry Writer
- Secret Manager Secret Accessor
